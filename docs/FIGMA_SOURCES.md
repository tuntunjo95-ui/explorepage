# Figma 设计稿索引

> 用途：集中记录 Pop Up 复刻可参考的 Figma 文件/Frame 链接。这里先存链接、状态和使用说明，不默认下载 `.fig`，也不默认实现未上线设计。

## 工作流约定

- 用户在工作电脑已登录设计账号后，把 **具体 Frame 链接 + 简单说明** 发给 Codex/Claude。
- 优先给选中 Frame 后复制的链接（带 `node-id`），不要只给整个文件首页链接。
- 不批量下载 `.fig` 本地副本：Figma 文件过大，且很多文件包含多代迭代版本，容易误用。
- 未上线或敏感设计默认只记录为“未上线，仅参考”，不实现、不部署、不保存图片，除非用户明确指定。
- 只有当前网站马上要校准/复刻的 Frame，才选择性保存参考截图或资源。
- 保存参考图时优先放在仓库内 `docs/figma_refs/`；只有代码实际引用的素材才放进 `src/assets/`。
- 使用某个 Figma 稿实现前，先读取该 Frame 的字体、字号、字重、行高、颜色、圆角、间距、截图和必要素材，再对照当前网页微调。

## 状态标签

- `当前可用`：可以用于当前线上复刻的字体、间距、颜色、组件校准。
- `未上线，仅参考`：只用于产品理解或后续规划，本轮不实现、不部署。
- `历史版本，谨慎用`：文件里有旧稿/迭代稿，必须按备注选择指定区域或最新版本。
- `待确认`：已记录，但还没确定是否用于当前网站。
